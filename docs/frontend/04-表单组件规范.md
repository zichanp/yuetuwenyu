# 表单组件规范

## 一、基础规则

所有后台表单必须使用 `el-form`。

表单项必须使用 `el-form-item`。

字段较多时，应使用卡片分组，不要把所有字段堆在一个大表单中。

## 二、字段展示规则

1. 必填字段必须配置校验规则。
2. 有默认值的字段，也需要清晰展示当前默认值。
3. 复杂字段应使用 `el-tooltip` 或说明文案解释。
4. 时间字段统一使用 `el-date-picker`。
5. 多行内容使用 `el-input type="textarea"`。
6. 开关型配置优先使用 `el-switch`。
7. 单选型配置优先使用 `el-radio-group`。
8. 多选型配置优先使用 `el-checkbox-group`。

## 三、按钮规则

表单底部按钮顺序：

```text
取消 / 返回 / 上一步 / 下一步 / 保存 / 提交
```

主要操作使用：

```text
type="primary"
```

普通操作使用默认按钮。

危险操作使用：

```text
type="danger"
```

## 四、校验规则

以下字段必须校验：

1. 活动名称。
2. 活动时间。
3. 报名时间。
4. 人数上限。
5. 手机号。
6. 身份证号。
7. 题目内容。
8. 选项内容。
9. 投票选项。
10. 上传文件。

## 五、错误提示

错误提示要说清楚用户应该怎么改。

推荐：

```text
请输入活动名称
请选择报名开始时间
报名结束时间不能晚于活动结束时间
人数上限必须为正整数
请至少添加 2 个投票选项
```

不推荐：

```text
不能为空
格式错误
参数异常
```
