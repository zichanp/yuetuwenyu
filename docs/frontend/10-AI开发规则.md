# AI 前端开发规则

## 一、强制规则

AI 在生成前端页面代码前，必须先阅读并遵循本目录下所有前端规范文档。

路径：

```text
docs/frontend/
```

## 二、开发前必须先输出

每次开发页面前，必须先输出以下内容：

1. 页面结构。
2. 使用的 Element Plus 组件。
3. 字段列表。
4. 交互说明。
5. 校验规则。
6. 接口字段建议。
7. 空状态。
8. 异常提示。
9. 二次确认逻辑。

确认后再生成代码。

## 三、组件使用规则

后台页面必须优先使用 Element Plus 组件。

禁止自行重复开发以下组件：

- Button
- Input
- Select
- Radio
- Checkbox
- DatePicker
- Form
- Table
- Pagination
- Dialog
- Drawer
- Tabs
- Card
- Tag
- Alert
- Tooltip
- Empty
- Message
- MessageBox

## 四、页面结构规则

后台管理页面统一采用：

```text
页面标题区
筛选查询区
操作按钮区
数据表格区
分页区
```

配置类页面统一采用：

```text
页面标题区
步骤条 / 分组卡片
表单配置区
底部操作区
```

数据概况页面统一采用：

```text
活动基础信息区
核心数据卡片区
趋势 / 分布区
明细入口区
空状态 / 异常提醒区
```

## 五、输出代码要求

生成代码时，请优先输出：

1. 页面文件。
2. 样式文件。
3. mock 数据。
4. 接口字段建议。
5. 后续需要后端配合的点。

## 六、禁止行为

1. 不要脱离 Element Plus 重新设计一套 UI。
2. 不要安装 Element UI。
3. 不要随意修改项目构建配置。
4. 不要删除已有业务代码。
5. 不要破坏已有路由、权限、接口逻辑。
6. 不要把 CDN 引入作为正式项目默认方案。
