# 弹窗与抽屉规范

## 一、弹窗使用场景

以下场景优先使用 `el-dialog`：

1. 新增。
2. 编辑。
3. 简单详情。
4. 二次确认。
5. 表单配置。
6. 导入导出操作确认。

## 二、抽屉使用场景

以下场景优先使用 `el-drawer`：

1. 复杂详情。
2. 报名详情。
3. 投稿详情。
4. 答题详情。
5. 投票详情。
6. 不希望离开当前列表页的详情查看。

## 三、二次确认规则

以下操作必须二次确认：

1. 删除。
2. 取消报名。
3. 驳回审核。
4. 清空配置。
5. 批量删除。
6. 批量导入。
7. 关闭重要开关。
8. 修改已生效规则。

## 四、确认文案要求

确认文案必须说明操作后果。

推荐：

```text
删除后数据不可恢复，是否确认删除？
取消报名后将释放该名额，是否确认取消？
关闭后用户将无法继续提交作品，是否确认关闭？
开启后将按新规则生效，是否确认开启？
```

不推荐：

```text
是否确认？
确定操作？
确认提交？
```
