# 空状态与异常提示规范

## 一、空状态

无数据时必须使用 `el-empty`。

常见文案：

```text
暂无数据
暂无报名记录
暂无投稿作品
暂无答题记录
暂无投票记录
暂无审核记录
暂无导入记录
暂无可展示的数据
```

## 二、异常提示

异常提示要说明原因和下一步操作。

推荐：

```text
当前活动暂无报名记录，请等待用户报名后查看。
当前筛选条件下暂无数据，请调整筛选条件后重试。
文件格式不支持，请上传 xlsx 或 csv 格式文件。
导入失败，请下载失败明细并修改后重新导入。
```

## 三、成功提示

成功提示要简洁。

推荐：

```text
保存成功
提交成功
导入成功
删除成功
操作成功
```

## 四、失败提示

失败提示要可理解。

推荐：

```text
保存失败，请稍后重试
导入失败，请检查文件格式
删除失败，该数据正在被使用
```
