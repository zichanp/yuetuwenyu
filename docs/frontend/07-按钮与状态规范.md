# 按钮与状态规范

## 一、按钮类型

| 场景 | 按钮类型 |
|---|---|
| 新建 | primary |
| 查询 | primary |
| 保存 | primary |
| 提交 | primary |
| 返回 | default |
| 重置 | default |
| 查看 | link |
| 编辑 | link |
| 导出 | default |
| 删除 | danger |
| 取消 | default 或 danger，视业务后果而定 |

## 二、按钮文案

按钮文案要短、明确。

推荐：

```text
创建活动
进入管理
查看详情
导出数据
取消报名
审核通过
审核驳回
```

不推荐：

```text
点击这里
确定
操作
处理
管理
```

## 三、状态展示

状态必须使用 `el-tag`。

状态文案要与业务一致，不要前后混用。

例如：

```text
未开始
进行中
已结束
待审核
审核通过
审核驳回
已签到
未签到
已签退
未签退
```

## 四、Element Plus 按钮注意事项

Element Plus 中列表操作按钮优先使用：

```vue
<el-button link type="primary">查看</el-button>
```

不建议继续使用旧写法：

```vue
<el-button type="text">查看</el-button>
```
