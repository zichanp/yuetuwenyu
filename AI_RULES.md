# Qoder 项目 AI 开发规则

## 一、项目技术栈

当前项目使用：

- Vue 3
- Element Plus

## 二、UI 规则

后台页面必须统一使用 Element Plus 组件库。

基础组件优先使用 Element Plus，不允许重复造轮子。

## 三、开发流程

每次开发页面时，请按以下流程执行：

1. 阅读 `docs/frontend` 下的前端规范。
2. 分析当前页面属于哪种页面类型。
3. 输出页面结构。
4. 输出 Element Plus 组件映射表。
5. 输出字段表。
6. 输出交互说明。
7. 输出校验规则。
8. 输出空状态和异常提示。
9. 再生成页面代码。

## 四、页面类型

### 列表管理页

统一结构：

```text
标题区
筛选区
操作区
表格区
分页区
```

### 新建 / 编辑配置页

统一结构：

```text
标题区
步骤条或分组卡片
表单配置区
底部操作区
```

### 数据概况页

统一结构：

```text
基础信息区
数据卡片区
趋势图表区
明细入口区
空状态区
```

## 五、代码要求

1. 代码必须能运行。
2. 不要破坏原有项目结构。
3. 不要删除已有文件。
4. 修改配置文件前先说明修改原因。
5. 新增依赖后请同步更新 `package.json`。
6. 页面样式应保持 Element Plus 后台管理系统风格。
7. 表单、表格、弹窗、状态、空状态必须使用 Element Plus 组件。
8. 列表操作按钮优先使用 `el-button link`。
